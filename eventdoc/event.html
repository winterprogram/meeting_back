Events documentation

Events to listen:-


  • Verify event
    The event ("verifyUser") listens on both the user and admin end to verify user and admin authentication. user and admin both will only be set as online user after verification of authentication token.
  • onlineUsersList
    This event ("onlineUsersList") has to be listened on the user's end to identify all available users that are currently online. All available users can be shown to the user based on which he can start chatting with a user. The output will be an object, object has key as userId and value as full name.
  • userId event
    The event ("userId") listens on user side . Whenever admin is creating any new meeting/updating/deleting any meeting scheduled at the same time user will get notification of the same and event is added to calender.
  • auth-error event
    The event ("auth-error") listens on both users and admin side when there is an error in authToken.

Events to emit:-


  • set-user event
    This event ("set-user") has to be emitted when normal user or admin get verify .
  • notify-updates events
    The event ("notify-updates") has to emit on every notification.
  • disconnect event
    This event ("disconnect") has to be emitted, when the user gets disconnected.
